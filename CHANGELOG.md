# Changelog

All notable changes to Tydal Radar are documented here.
Format follows [Keep a Changelog](https://keepachangelog.com/); this project is pre-1.0.

## [Unreleased]

### Added
- **Phase 1 built — Engineering + Revenue (the first hires)**: the org's first four agents. **Seam generalized to "one meter, two doors"** — `lib/agentLog.ts` (shared `runCostColumns` + `scheduleBudgetAlert`) so in-process runs (`runAgent`) and **external** runs write identical `agent_runs` rows. External Engineering agents (Claude Code in GitHub Actions) report via two token-gated endpoints — `GET /api/ops/precheck` (budget gate) and `POST /api/ops/ingest` (log a completed run) — reusing `costOf`/budgets/alerts. **Approvals queue** (the slot Phase 0 left in `/ops`): `agent_tasks` + a "Needs your yes" lane (PRs to merge + escalations, resolvable via `/api/ops/tasks`); `/api/ops/summary` gained `pending` + a trigger chip. **Revenue:** `lib/pitchAgent` (shared Claude call) + `/api/ai/pitch/batch` (capped, budget-aware pre-draft of the day's route) + the **Pipeline Steward** daily cron (`/api/ops/steward` + `lib/steward` — going-cold/overdue selection, a Haiku digest through `runAgent`, escalation tasks for going-cold quotes, and a push). **Engineering runners:** role briefs (`docs/agents/*.md`) + Actions workflows (`agent-maintenance.yml` weekly, `agent-feature.yml` on-demand) that run Claude Code → gate → PR *you* merge → ingest cost. No outbound messages; no agent pushes to `main`. Pure cores unit-tested. **One-time human steps**: run the `agent_tasks` migration (`docs/phase-1-eng-revenue.md` §4), set `OPS_INGEST_SECRET` + Actions `ANTHROPIC_API_KEY` + `APP_BASE_URL`, seed the §7 dept/role budgets, and dry-run the workflows.
- **Phase 0 built — Observability & Budgets (the CFO layer)**: every AI call now flows through one instrumented seam, `runAgent()` (`lib/runAgent.ts`) — budget pre-check → run → capture token usage → compute cost (`lib/pricing.ts`, stored on the row) → log to `agent_runs` → post-response threshold push. Budget enforcement (`lib/agentBudget.ts`) blocks a run when its global/department/role cap is hit for any period; Web Push (`lib/serverPush.ts`, reusing the follow-up VAPID + `push_subscriptions`) fires once when spend crosses 80%/100% of a cap. First instrumented employee: `/api/ai/pitch` (role `pitch-writer` / dept `revenue`; over-cap → HTTP 429). Gated cockpit at **`/ops`** — spend cards, budget bars (green → amber ≥80% → red ≥100%), agents running now, recent runs, error feed, and top spenders — reading `GET /api/ops/summary` (gated, service-role, returns an empty cockpit if the tables aren't migrated). Cost/budget math is unit-tested. **One-time human step**: run the `agent_runs` + `agent_budgets` migration (RLS-locked, seed caps) from `docs/phase-0-observability.md` §2.
- **Phase 0 spec — Observability & Budgets** (`docs/phase-0-observability.md`): the CFO layer for the AI org — an `agent_runs` + `agent_budgets` data model (RLS-locked), a `costOf` pricing module, a `runAgent` instrumentation seam (budget pre-check → run → capture usage → cost → log → alert), budget enforcement (org-level spawn cap + per-run token cap), Web Push threshold alerts, and a gated cockpit dashboard (`/api/ops/summary` + `/ops`). Buildable, with a step-by-step checklist.
- **AI-organization blueprint** (`docs/ai-org.md`): the master plan for running Tydal's internal operations as an AI-staffed org — doctrine, org chart, ~18 narrow roles across six departments (junior→senior→director→chief), the fan-out doctrine (roles × on-demand instances), the context-economy toolkit, cost tiers, a dominance scoreboard (KPIs), clean-win/legal guardrails (CASL, Law 25), and a phased rollout starting with observability. Blueprint only — nothing auto-runs yet.
- **AI pitch (Claude)**: on the prospect card, "✨ Draft with AI" generates a grounded, **bilingual (Québec French + English)** door pitch — opener, likely objections + responses, who to ask for, and the strongest angle — with an FR/EN toggle (French default). Served via `/api/ai/pitch` (key server-side, behind the password gate), cached per prospect in `localStorage` (instant/offline), falling back to the templated pitch when unconfigured. Defaults to `claude-sonnet-5`. See docs/ai-pitch.md. The card also drops the abstract lead-score chip and the corridor hint to keep the pitch the focus.
- **Market & sales underwriting engine** (`lib/underwriting.ts`): scores every prospect on expected value per unit of effort — `P(win) × contract value × timing × confidence ÷ effort`. The Search hot list now ranks by this (folding in travel from your location), the prospect card shows a $/$$/$$$ value band, and "Plan my walk" picks the highest-EV unworked doors nearby before routing them.
- **Playbook on the prospect card**: best-time-to-knock by business type, a grounded (templated) door opener, and supporting angles built from the prospect's own signals (newly opened, incumbent, sole occupant, rating, urgency).
- **Same-type corridor hint**: "N more [type] within ~400 m — pitch them in one loop."
- **Stale-deal leakage**: Follow-ups shows a "Going cold" section for active deals (knocked/talked/quoted) left untouched 14+ days.
- **Expansion graph**: on a client's card, warm expansion paths — for a multi-tenant building, a prompt to pursue the property manager for the rest of the building (consistent with the decision-maker model, rather than pitching each tenant); plus sister locations of the same chain.
- Time-series growth signals (the market-intel moat): the monthly snapshot now records review counts, and `diff-snapshots.mjs` reports "just opened" businesses (no cleaning contract yet), "gone / likely closed", and "fastest growing" by review momentum. In-app, a `first_seen` column powers a "✦ New" badge on the prospect card and a +20 lead-score boost for newly-opened businesses (requires the `first_seen` migration — see below).
- Co-location / decision-maker signal: prospects are grouped by building, so the card shows whether a business is a **sole occupant** (controls its own cleaning — a direct-pitch target) or **shares its address** with others (cleaning likely handled by a property manager — a different motion). ~75% of prospects are sole occupants.
- Automated market refresh (local, monthly): `scripts/refresh-and-report.sh` + a launchd job re-scrape the full city, then diff the two newest snapshots into a "+N new · -N gone" report with a Notification Center ping. The scraper now writes a per-run `market-snapshots/` found set (reveals openings **and** closures, unlike DB backups which only show growth); `diff-snapshots.mjs` defaults to those. See docs/market-refresh.md.
- Lead scoring / "hot list": a 0–100 `leadScore` (opportunity by stage + size/quality from reviews & rating + timing from contract/follow-up urgency + website) plus the underwriting engine rank prospects by who to work next. Search's empty state shows the top prospects, ranked by expected value, each tagged with a $/$$/$$$ value band; the colour-coded lead-score badge appears on the prospect card.
- Prospect enrichment: the scraper + in-app Refresh now capture Places `rating`, review count, and website, shown on the prospect card. Backfill existing rows with `node --env-file=.env.local radar-grid-scrape.js --update` (a re-scrape that overwrites instead of insert-only).
- Market-intelligence analytics: a "Market coverage" table (size / worked / open per ICP segment — surfaces whitespace) and a "Competitors" breakdown (incumbents from `current_provider`) in the Stats sheet.
- Market change over time: `scripts/diff-snapshots.mjs` diffs two backup snapshots to show businesses added (growth) vs gone (churn/closed), with a per-type breakdown.
- Follow-up push notifications (opt-in): a "Remind me" toggle in Follow-ups subscribes the device to Web Push; a daily Vercel Cron pushes "N follow-ups due today" so reps act without opening the app. Requires one-time VAPID/Supabase setup — see docs/push-setup.md.
- Route planning ("Plan my walk"): a map control that orders your nearest unworked (not-knocked) prospects into an efficient nearest-neighbour walking sequence — numbered stops with per-leg distance, total distance, tap-to-open, and per-stop Directions.
- Canvassing efficiency (uses your GPS location, one shared watcher): Search results sort nearest-first; distance to each prospect shows on Search/Follow-up rows and the prospect card; a "Near me · not knocked" one-tap filter surfaces unworked prospects within ~1 km; and tapping a stage chip on the card now saves+closes in one tap (except "lost", which still asks for a reason).
- Accessibility: the Search / Follow-ups / Contracts / Stats pull-up sheets and the prospect card are now proper labelled dialogs (`role="dialog"`, `aria-modal`, `aria-label`) and dismiss on the Escape key.
- Offline-first: prospects/pipeline are mirrored in IndexedDB and render instantly (stale-while-revalidate when online); prospect edits save optimistically to a local outbox and sync to Supabase on reconnect/foreground, so edits made in a dead zone are never lost. A top-left status pill shows offline / pending-count / syncing state (hidden when synced).
- Citywide grid scraper (`radar-grid-scrape.js`) — tiles Montreal into ~2.2 km cells to beat Google Places' 60-result/search cap; `--only=<types>` flag for targeted runs.
- `medical` ICP type (clinics/doctors), distinct from professional `office`.
- `veterinary` ICP type.
- Supabase keep-alive (Vercel Cron `/api/keepalive`, every 3 days) to prevent free-tier auto-pause.
- Multi-select ICP type filter in FilterPanel.

### Changed
- Security (lockdown): prospect + pipeline **reads and writes now go through password-gated server routes** (`/api/data/*`) using the service-role key, so the public anon key can be denied all access to those tables (run the SQL in docs/security-writes.md after deploy). `keepalive` moved to the service-role client too. Push subscriptions remain anon (separate, low-sensitivity table).
- `office` now means professional services (lawyer / accounting / real estate), merged into one bucket.

### Fixed
- Map could pan/zoom into the empty world beyond the map: added a `restriction` to greater Montréal (`strictBounds`) plus a `minZoom` floor, so zooming out or panning now stops at the region border.
- Hydration mismatch on every load: `isOnline()` used `typeof navigator === 'undefined'` to detect the server, but Node 18+ defines a global `navigator` without `onLine`, so the server rendered the offline `SyncStatus` pill while the client rendered nothing — React then threw away and regenerated the tree. Now guards on `navigator.onLine` being a boolean.
- Flaky dev server: a stray parent-dir lockfile made Turbopack infer the wrong workspace root and mis-resolve modules (e.g. `@mui/material-nextjs`) after cache invalidation. Pinned `turbopack.root` in `next.config.ts`.
- iOS PWA map dead-zone: the map now flex-fills the space above the tab bar.
- Prospect fetch loaded only the first 1000 rows (PostgREST cap); now paginated.
- Notes/fields in the prospect sheet sat behind the iOS keyboard; now scrolled into view above it.
